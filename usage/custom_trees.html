Custom Trees

Constructing an Additive Tree Ensemble or AddTree

Veritas uses its own tree ensemble representation. You can manually build one to try Veritas out. Here’s an example of a manually constructed tree ensemble.

Python
 1import numpy as np
 2from veritas import *
 3
 4# Manually create a two-tree ensemble
 5#
 6#       F0 < 2                     F0 < 3
 7#       /    \                     /    \
 8#   F0 < 1   F0 < 3     +     F1 < 5     F1 < 0
 9#   /   \     /   \           /   \       /    \
10#  3     4   5     6         30   40     50     F2
11#                                             /    \
12#                                            70    80
13
14at = AddTree(1)  # Empty ensemble with int his case 1 value in the leafs
15t = at.add_tree();
16t.split(t.root(), 0, 2)   # split(node_id, feature_id, split_value)
17t.split( t.left(t.root()), 0, 1)
18t.split(t.right(t.root()), 0, 3)
19t.set_leaf_value( t.left( t.left(t.root())), 3)
20t.set_leaf_value(t.right( t.left(t.root())), 4)
21t.set_leaf_value( t.left(t.right(t.root())), 5)
22t.set_leaf_value(t.right(t.right(t.root())), 6)
23
24t = at.add_tree();
25t.split(t.root(), 0, 3)
26t.split( t.left(t.root()), 1, 5)
27t.split(t.right(t.root()), 1, 0)
28t.split(t.right(t.right(t.root())), 2) # Boolean split (ie < 1.0)
29t.set_leaf_value( t.left( t.left(t.root())), 30)
30t.set_leaf_value(t.right( t.left(t.root())), 40)
31t.set_leaf_value( t.left(t.right(t.root())), 50)
32t.set_leaf_value( t.left(t.right(t.right(t.root()))), 70)
33t.set_leaf_value(t.right(t.right(t.right(t.root()))), 80)
34
35# Print the trees (including the node-ids)
36print(at[0])
37print(at[1])
38
39# Evaluate this ensemble
40print("Eval:", at.eval(np.array([[0, 0, 0], [15, -3, 9]])))

This outputs the following. Note that the Boolean split on feature 2 is replaced with a less than split splitting on value 0.5 (veritas.BOOL_SPLIT_VALUE). You can use the pre-defined domains for TRUE and FALSE: veritas.TRUE_DOMAIN and veritas.FALSE_DOMAIN.

Model Conversion implementation

Converting representations of other learners or your own models should be easy and can be done by implementing the class AddTreeConverter. In the following example MyAddTreeConverter implements the get_addtree method from AddTreeConverter for a trivial tree representation. The trees consist of a boolean split in the root with only 2 leaves. After adding an instance of MyAddTreeConverter to the convertermanager, the same method get_addtree that was used in the previous example can be used for the new model representation aswell as the previously methoned ones.

Python
 1# Trivial Tree representation
 2#
 3#         F0             F0
 4#       /    \    +    /    \
 5#      10    20       12    13
 6
 7myModel = [[10,20,0],[12,13,0]] # [left leaf, Right leaf, Boolean Feature]
 8
 9class MyAddTreeConverter(AddTreeConverter):
10    def get_addtree(self,model):
11        # Implement AddTreeConverter using your own model
12        addtree = AddTree(1)
13
14        for tree in model:
15            t = addtree.add_tree()
16            t.split(t.root(),1)
17            t.set_leaf_value(t.left(t.root()),tree[0])
18            t.set_leaf_value(t.right(t.root()),tree[1])
19
20        return addtree
21
22
23# Add converter instance to the converter_manager
24add_addtree_converter(MyAddTreeConverter())
25
26# Use get_addtree() on your own models
27addtree = get_addtree(myModel)
28
29print(f"{addtree}\n")
30
31print(addtree[0])
32print(addtree[1])
